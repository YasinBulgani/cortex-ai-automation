Faz 0 — Ücretsiz AI Entegrasyonu (BAŞLANGIÇ NOKTASI)

⏱ 2-3 gün📁 Engine + Backend🎯 Tüm akışın temeli
KRİTİK İLK▼

GROQ ENTEGRASYONU

  • Groq ücretsiz API key alımı (console.groq.com)
  • groq-sdk paketi kur (Python + Node)
  • llama3-70b-8192 model ile test
  • Mevcut ai_engine.py'e Groq provider ekle
  • Hız testi: token/saniye ölç

GEMINI ENTEGRASYONU

  • Google AI Studio'dan ücretsiz Gemini key al
  • google-generativeai paketi kur
  • gemini-1.5-flash model — hızlı ve ücretsiz
  • Fallback provider olarak ekle
  • Rate limit: dakikada 60 istek

OLLAMA (LOKAL MODEL)

  • ollama.ai'den kur (macOS native)
  • llama3:8b veya mistral:7b indir
  • Engine'e Ollama provider ekle
  • İnternet bağlantısı olmadan çalışır

AI GATEWAY MİKROSERVİS

  • Yeni FastAPI servisi: ai-gateway/ klasörü
  • Model routing: Groq → Gemini → Ollama → g4f
  • Rate limit tespiti + otomatik fallback
  • Retry mekanizması (3 deneme, en iyi seç)
  • Nginx'e /api/ai/* proxy ekle
  • docker-compose'a ai-gateway servisi ekle

🤖 Model Öncelik Zinciri

Groq (llama3-70b) → Gemini 1.5 Flash → Ollama (local) → g4f (fallback)
Rate limit aşılırsa otomatik sonraki modele geç. Hepsi çalışmazsa offline mod.

Tahmini maliyet: $0 — tamamen ücretsiz
engine/core/ai_engine.py ai-gateway/ (YENİ MİKROSERVİS) ai-gateway/providers/groq_provider.py ai-gateway/providers/gemini_provider.py ai-gateway/providers/ollama_provider.py infra/nginx/nginx.conf Makefile (make dev komutu)

Faz 2 — Doküman Yükleme & AI Parse Pipeline

⏱ 3-4 gün📁 Backend + AI Gateway + Frontend🎯 Temel AI akışı
KRİTİK▼

DOKÜMAN YÜKLEYİCİ

  • Word (.docx) parser — python-docx ile metin çıkar
  • PDF parser — PyMuPDF ile metin + tablo çıkar
  • Confluence URL çekici — Confluence REST API (API token)
  • MinIO'ya yükle, DB'de referans sakla
  • Büyük doküman: otomatik chunk (3000 token/parça)

AI DOKÜMAN ANALİZİ

  • Serbest format dokümanı AI'a gönder
  • AI çıkarımı: iş alanı, kullanıcı rolleri, temel akışlar
  • Modül tespiti: hangi modüller var? (Login, Ödeme, Rapor...)
  • Stream output ile canlı analiz gösterimi
  • Confidence score her çıkarım için

DB BAĞLANTISI (OPSİYONEL)

  • DB bağlantı bilgisi şifreli sakla (AES encrypt)
  • PostgreSQL, MySQL, Oracle, MSSQL connector'ları
  • 100+ tablo varsa modüle ayır, grup grup analiz
  • DB şeması + doküman birleştir → zengin bağlam

BACKEND ENDPOINTLERİ

  • POST /api/v1/documents/upload (Word/PDF)
  • POST /api/v1/documents/confluence-fetch
  • POST /api/v1/documents/{id}/analyze (AI)
  • GET /api/v1/documents/{id}/analysis-result
  • WebSocket /ws/analysis-progress/{jobId}

🤖 AI Kullanım Detayı — Faz 2

Doküman içeriği → chunklara bölünür → her chunk AI'a gönderilir → sonuçlar birleştirilir → modül haritası çıkar.
Büyük dokümanlarda özet önce çıkarılır, özet üzerinden analiz yapılır.

Model: Groq llama3-70b (hız için) | Fallback: Gemini 1.5 Flash
backend/app/domains/documents/ (YENİ) backend/app/domains/datasets/ (YENİ) ai-gateway/prompts/document_analysis.py app/(dashboard)/p/[id]/analysis/page.tsx

Faz 3 — AI Test Case Üretimi & Toplu Onay

⏱ 4-5 gün📁 Backend + Engine + Frontend🎯 Temel ürün değeri
KRİTİK▼

AI TEST CASE ÜRETİCİ

  • Bağlam: doküman + DB şeması + geçmiş sonuçlar + kullanıcı yorumları
  • Gherkin formatında (Türkçe/İngilizce — ayardan)
  • Scenario Outline — mümkün olan her yerde
  • Kapsam: Fonksiyonel + Negatif + Güvenlik + Performans
  • AI ID üretimi: anlamlı, mod ül tabanlı
  • AI öncelik önerisi: P0/P1/P2/P3
  • AI tag önerisi: @smoke @regression @critical vs.

TOPLU ONAY EKRANI

  • Tüm AI önerilerini tek listede göster
  • Toplu seç + onayla / reddet
  • Her test case için düzenleme modu
  • Öncelik değiştirme, tag ekleme/çıkarma
  • 'Detay' butonu — AI neden bu testi önerdi?
  • Onaylananlar DB'ye kaydedilir

TEST CASE YÖNETİMİ

  • Metin + filtre + AI doğal dil arama
  • Sabit + serbest + AI otomatik tag sistemi
  • Versiyon takibi — her değişiklik kaydedilsin
  • Versiyonlar arası diff görünümü
  • Soft delete — test case arşivlensin

TEST VERİSİ

  • DB bağlıysa: örnek satırlar çek, maskele
  • DB yoksa: AI domain'e göre sentetik üretsin
  • Scenario Outline Examples tablosu otomatik doldur
  • Excel/PostgreSQL'e kaydet

🤖 AI Kullanım Detayı — Faz 3

Analiz sonucu + DB şeması → Test case prompt → AI JSON output (id, title, tags, priority, gherkin, confidence)
3 retry: en yüksek coverage score'lu seçilir. Kullanıcı onaylarsa DB'ye kaydedilir.

Model: Groq llama3-70b | JSON mode | Temperature: 0.3
backend/app/domains/testcases/ (YENİ) ai-gateway/prompts/test_generation.py app/(dashboard)/p/[id]/scenarios/page.tsx components/bulk-approval.tsx

Faz 4 — AI Regresyon Seti Önerisi

⏱ 3-4 gün📁 Backend + Frontend🎯 Akıllı test seçimi
YÜKSEK▼

REGRESYON ÖNERİCİ

  • Doküman değişikliği → etkilenen modüller → ilgili testler
  • DB tablo bağımlılık grafiği (FK analizi)
  • Risk skoru hesaplama her test için
  • AI: "Bu alanlar değişti → şu testler etkilenir" gerekçesi

TETIK MEKANİZMASI

  • Manuel: "Regresyon Güncelle" butonu
  • Yeni doküman yüklenince otomatik güncelle
  • DB şema değişikliği tespiti → otomatik güncelle
  • Her deploy'da güncel regresyon seti koşulsun

REGRESYON SET YÖNETİMİ

  • Öncelik görünümü: P0 → P1 → P2 → P3
  • Tahmini süre hesabı (test sayısı × ortalama süre)
  • Bağımlılık grafiği görselleştirme (D3.js)
  • Set geçmişi — hangi sette ne vardı?

DB MODELLERİ

  • RegressionSet tablosu
  • RegressionSetItem tablosu (test_case_id, priority, reason)
  • RegressionSetHistory — her güncelleme kaydı

🤖 AI Kullanım Detayı — Faz 4

Değişiklik diff + test case listesi → AI bağımlılık analizi → öncelikli regresyon seti JSON çıktısı.
Her test için: "neden regresyonda?" gerekçesi + güven skoru.

Model: Groq llama3-70b | Reasoning modu | Detaylı prompt

Faz 5 — Otomasyon Kodu Üretimi (NexusQA + Playwright)

⏱ 4-5 gün📁 Engine + NexusQA + Frontend🎯 Otomasyonun kalbi
YÜKSEK▼

SEÇIM EKRANI

  • Test case listesinde checkbox'lar
  • AI otomasyon uygunluk skoru badge'i (%92 gibi)
  • "AI önerilerini seç" butonu — yüksek skor olanları seç
  • Framework seçimi: Java/Selenium veya Playwright TS
  • Toplu "Otomasyona Çevir" butonu

JAVA/SELENIUM KOD ÜRETİCİ (MAVİYAKA)

  • Gherkin feature dosyası → features/ai-generated/ klasörüne yaz
  • Önce NexusQA mevcut step'lerini eşleştir (ClickSteps, InputSteps vb.)
  • Eksik step varsa yeni Java step definition yaz
  • POM: URL tarama + locator keşfi + otomatik class üretimi
  • Derleme kontrolü: mvn compile ile doğrula
  • Lint: Checkstyle ile stil kontrol
  • AI self-review skoru hesapla

PLAYWRIGHT KOD ÜRETİCİ

  • Gherkin → TypeScript Playwright kodu
  • Page Object Model otomatik üretimi
  • Her koşuda video kaydı yapılandırması
  • ESLint + TypeScript derleme kontrol
  • API testi: Supertest entegrasyonu

TEST VERİSİ ENTEGRASYONİ

  • DB bağlıysa örnek satır çek + maskele
  • Yoksa AI sentetik veri üretsin
  • Scenario Outline Examples tablosu doldur
  • Excel dosyası otomatik oluştur

🤖 4 Aşamalı AI Kod Üretim Zinciri

1️⃣ Manuel test adımları → Gherkin feature (AI)
2️⃣ Gherkin → Step Definitions (AI, NexusQA step'lerini önce kontrol eder)
3️⃣ Step Defs → Playwright/Java kodu (AI)
4️⃣ Üretilen kod → Self-Review + Skor (AI)

Model: Groq mixtral-8x7b (kod üretimi için daha iyi) | Fallback: Gemini 1.5 Flash
engine/routes/pipeline_routes.py (GENİŞLET) engine/routes/batch_automation_routes.py (YENİ) NexusQA/features/ai-generated/ (ÇIKTI) ai-gateway/prompts/code_generation.py components/automation-selector.tsx

Faz 6 — Test Koşusu, Allure & AI Debug

⏱ 3-4 gün📁 Engine + Frontend + WebSocket🎯 Tam döngü kapanışı
YÜKSEK▼

TEST KOŞUSU

  • Platform mvn test subprocess olarak çalıştırır
  • WebSocket ile canlı log stream'i UI'a
  • İlerleme çubuğu: kaç test bitti / kaç kaldı
  • Video kaydı her koşuda otomatik
  • Koşu tamamlanınca tarayıcı push notification

ALLURE + PLATFORM RAPORU

  • Allure detay raporu: her test adımı, screenshot, video
  • Platform dashboard özeti: pass/fail oranı, trend grafiği (D3.js)
  • Koşu geçmişi: her koşu kaydedilsin
  • Karşılaştırma: önceki koşu vs mevcut

AI DEBUG & ANALİZ

  • Fail olan testler → AI analiz: neden fail oldu?
  • Flaky test tespiti: aynı test bazen geçti bazen geçmedi
  • Bug mı, flaky mi? AI karar önerisi (ben karar veririm)
  • Düzeltme önerisi: "Şu locator değişmiş, güncelle"
  • Self-healing: broken locator otomatik onar

FEEDBACK LOOP

  • Test sonuçları → regresyon setini güncelle
  • Sık fail olanlar → öncelik artır
  • Hiç fail olmayanlar → regresyondan çıkar?
  • AI öğrenme sinyali: onay/red kaydet

🤖 AI Debug Akışı

Fail log + screenshot + Allure output → AI analiz → "Element bulunamadı: #submit-btn, selector değişmiş olabilir" + düzeltme önerisi.
Kullanıcı "Bu bug" veya "Flaky" seçer, AI öğrenir.

Model: Groq llama3-70b | Log analizi modu
