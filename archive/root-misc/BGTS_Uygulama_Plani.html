Faz 2 — DB Bağlantı & Analist Dokümanı Pipeline

⏱ 4-5 gün 📁 Backend + AI Engine + Frontend 🎯 Projenin kalbi
KRİTİK

Yapılacaklar

  • ☐ DB Connector Service: PostgreSQL, MySQL, Oracle, MSSQL bağlantı desteği. Kullanıcı connection string girer → tablolar, kolonlar, ilişkiler, indexler keşfedilir
  • ☐ Schema Analyzer: DB yapısını JSON'a çevir — tablo sayısı, kolon tipleri, FK ilişkileri, veri örnekleri (ilk 5 satır)
  • ☐ AI Analist Doküman Üretici: Schema JSON → LLM'e gönder → Yapılandırılmış analist dokümanı:
    — İş alanı tanımı (domain analysis)
    — Kritik tablolar ve ilişkileri
    — Olası iş kuralları (business rules inference)
    — Önerilen test alanları (suggested test areas)
    — Risk analizi (hangi tablolar en çok değişir, en çok ilişkisi olan)
  • ☐ Backend Endpoints:
    — POST /api/v1/datasets/connect → DB bağlantı testi
    — POST /api/v1/datasets/analyze-schema → Schema analiz
    — POST /api/v1/analyst-documents/generate → AI doküman üretimi
    — GET /api/v1/analyst-documents/{id} → Doküman görüntüleme
  • ☐ DB Modelleri: AnalystDocument, DatabaseConnection, SchemaSnapshot tabloları
  • ☐ Frontend: DB bağlantı formu, schema tree görselleştirme, analist dokümanı önizleme/düzenleme sayfası

AI Entegrasyon Detayı

Model: Claude Sonnet 4 / GPT-4o (kullanıcı seçimi)
Prompt Stratejisi: Schema JSON + domain context → structured output (JSON mode)
Token Yönetimi: Büyük şemalar için chunking (tablo grupları halinde)
Output: Markdown analist dokümanı + JSON metadata

Dokunulacak Dosyalar

backend/app/domains/datasets/ (YENİ) backend/app/domains/analyst/ (YENİ) backend/app/domains/ai/service.py backend/app/infra/models.py app/(dashboard)/p/[projectId]/analysis/page.tsx app/(dashboard)/p/[projectId]/veri-kaynagi/page.tsx

Faz 3 — AI ile Test Case & Regresyon Seti Önerisi

⏱ 4-5 gün 📁 Backend + Engine + Frontend 🎯 Test tasarım otomasyonu
KRİTİK

Yapılacaklar

  • ☐ Test Case Generator: Analist dokümanını al → AI ile manuel test case'leri üret:
    — Pozitif senaryolar (happy path)
    — Negatif senaryolar (boundary, invalid)
    — Veri bütünlüğü testleri (FK, unique constraint)
    — Performans risk alanları
    — Güvenlik kontrolleri (SQL injection, auth)
  • ☐ Regresyon Seti Önerici: AI risk bazlı önceliklendirme:
    — "Bu tabloda değişiklik olursa şu testler koşmalı"
    — Bağımlılık haritası (dependency graph)
    — Öncelik skorlaması (P0-kritik, P1-yüksek, P2-orta, P3-düşük)
  • ☐ Backend Endpoints:
    — POST /api/v1/test-cases/generate-from-analysis
    — POST /api/v1/regression-sets/suggest
    — PUT /api/v1/test-cases/{id}/approve (kullanıcı onayı)
    — GET /api/v1/regression-sets/{projectId}
  • ☐ DB Modelleri: GeneratedTestCase, RegressionSet, RegressionSetItem tabloları
  • ☐ Frontend: Test case listesi (düzenlenebilir), regresyon seti görselleştirme (dependency graph), onay/red butonları

AI Entegrasyon Detayı

Model: Claude Sonnet 4 / GPT-4o
Input: Analist dokümanı + Schema bilgisi + (opsiyonel) önceki test sonuçları
Output: Yapılandırılmış test case JSON listesi (Gherkin-ready format)
Regresyon: Tablo bağımlılık grafiği + risk skoru hesaplama
Feedback: Kullanıcı onay/red → model fine-tuning sinyali

Faz 4 — Otomasyon Seçimi & Toplu Kod Üretimi

⏱ 3-4 gün 📁 Engine + Frontend 🎯 Manuel → Otomasyon köprüsü
YÜKSEK

Yapılacaklar

  • ☐ Seçim UI: Test case listesinde checkbox'lar → "Seçilenleri Otomasyona Çevir" butonu
    — AI önerisi: "Bu test otomasyona uygun (%92 güven)" badge'leri
    — Toplu seçim (Tümünü seç / AI önerilerini seç)
  • ☐ Batch Code Generator: Seçilen test case'leri sırayla:
    1. Manuel test → Gherkin feature dosyası (AI)
    2. Gherkin → Step definitions (AI)
    3. Step definitions → Playwright test kodu (AI)
    4. Page Object Model otomatik üretimi (AI)
    — Progress bar ile takip
  • ☐ Engine Endpoint:
    — POST /api/pipeline/batch-generate → Job ID döner
    — GET /api/pipeline/batch-status/{jobId} → İlerleme takibi
    — GET /api/pipeline/batch-result/{jobId} → Üretilen dosyalar
  • ☐ Mevcut pipeline_routes.py Genişletme: Tek test → toplu test desteği
  • ☐ Üretilen Kod Kalitesi: AI self-review (üretilen kodu tekrar AI'a gönder → hata tespiti → düzeltme)

AI Entegrasyon Detayı

Zincir: Manuel Test → [AI] → Gherkin → [AI] → Step Def → [AI] → Playwright → [AI] → Self-Review
4 aşamalı AI pipeline — her aşamada farklı system prompt
Self-Healing: Üretilen locator'lar çalışmazsa AI ile otomatik düzeltme
POM: Sayfa bazlı Page Object class'ları otomatik üretim

AI/LLM Kullanım Haritası — Her Adımda AI

1. Schema Analizi

DB tablolarını, ilişkilerini, iş kurallarını AI ile analiz et. Domain keşfi otomatik.

Claude Sonnet 4 / GPT-4o | Structured Output

2. Analist Dokümanı Üretimi

Schema analizinden kapsamlı test analiz dokümanı oluştur. İş kuralları, risk alanları, test stratejisi.

Claude Opus 4 / GPT-4o | Long Context

3. Test Case Üretimi

Analist dokümanından pozitif/negatif/boundary test case'leri otomatik üret.

Claude Sonnet 4 / GPT-4o | JSON Mode

4. Regresyon Seti Önerisi

Değişiklik etkisi analizi, bağımlılık grafiği, risk bazlı önceliklendirme.

Claude Sonnet 4 | Reasoning

5. Otomasyon Kodu Üretimi

Manuel test → Gherkin → Step Definition → Playwright kodu. 4 aşamalı zincir.

Claude Sonnet 4 / GPT-4o | Code Generation

6. Self-Healing & Review

Üretilen kodu otomatik review et, kırık locator'ları onar, kalite skoru hesapla.

Claude Sonnet 4 | Code Review

7. Chat Asistan

Proje context'i ile soru-cevap. "Bu test neden fail oldu?" "Hangi testler etkilenir?"

Claude Sonnet 4 / GPT-4o | Conversational

8. Feedback & Öğrenme

Test sonuçlarından pattern çıkar, gelecek önerileri iyileştir, flaky tespiti yap.

Tüm modeller | Analytics
