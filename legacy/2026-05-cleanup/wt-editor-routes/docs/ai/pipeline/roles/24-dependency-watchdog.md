# 24 · Dependency Watchdog

**Slug:** `dependency_watchdog`  
**Tip:** Out-of-pipeline (scheduled)  
**Trigger:** Günlük (cron), GitHub advisory webhook  
**Çıktı:** CVE bulguları için `BUG-<NNN>`, lib güncellemeleri için `FEAT-<NNN>` veya `GAP-<NNN>`

---

## Amaç

Bağımlılıkları otomatik izle: CVE var mı, eskiyen package var mı, breaking semver güncellemesi var mı? Bulgulardan pipeline item'ı aç, intake'i atla (doğrudan pipeline'a gir).

Bu rol senin mevcut `fix/security-critical-patches` branch tipini **proaktif** hale getirir.

---

## Başlama tetikleyicisi

- **Günlük cron:** 02:00 UTC, GitHub Actions üzerinden
- **Event:** GitHub Dependabot/GHSA advisory webhook
- **Manuel:** `make pipeline-deps-scan`

---

## Input

1. `backend/requirements.txt`, `backend/requirements-dev.txt`
2. `apps/web/package.json` + `package-lock.json`
3. `engine/requirements.txt`
4. `Dockerfile*` ve `docker-compose*.yml` image tag'leri
5. GitHub Advisory Database

---

## Work

1. **Scan koş**:
   ```bash
   # Python
   cd backend && pip-audit --format json -r requirements.txt > /tmp/pip-audit.json
   cd engine && pip-audit --format json -r requirements.txt > /tmp/engine-audit.json
   # JS
   cd apps/web && npm audit --json > /tmp/npm-audit.json
   # Container image
   trivy image <image> --format json > /tmp/trivy.json
   ```
2. **CVE sınıflandırma**:
   - CVSS ≥ 9 (critical) → **BUG** critical priority, acil pipeline
   - CVSS 7-8.9 (high) → **BUG** high priority
   - CVSS 4-6.9 (medium) → **GAP** medium priority (batch)
   - CVSS < 4 (low) → log, batch monthly
3. **Duplicate kontrolü**: aynı CVE için zaten item var mı? (state.json'da başlık match)
4. **Lib upgrade check** (non-security):
   - Eskimiş major version (2+ behind) → `FEAT` ile upgrade item
   - Breaking change announced → pre-emptive GAP
5. **Otomatik PR**: basit patch varsa (security patch non-breaking) → renovate-style otomatik PR aç
   - Not: otomatik PR de pipeline'dan geçer (security_reviewer bu PR için de çalışır)
6. **Item aç**:
   ```bash
   stage.sh init BUG "CVE-2026-XXXXX — <package> <version> [<severity>]" \
     --scope fe=false,be=true,data=false,infra=false,perf_sensitive=false
   ```
7. **Rapor commit** (haftalık rollup):
   ```
   docs/ai/dep-scans/YYYY-MM-DD.md
   ```

---

## Output

- N yeni pipeline item'ı (CVE'lere göre)
- Weekly scan raporu
- Otomatik upgrade PR'ları (uygun durumda)

---

## Done kriteri

- ✅ Tüm dep dosyaları tarandı
- ✅ CVE'ler sınıflandırıldı ve item'a dönüştürüldü
- ✅ Duplicate açılmadı
- ✅ Weekly rapor yazıldı

---

## Yasaklar

1. Critical CVE'yi batch'e koyma (her biri ayrı BUG, acil)
2. False positive'leri görmezden gelme (dokümante et, `.audit-ignore`)
3. Otomatik major version upgrade (breaking!) — sadece patch/minor otomatize
4. Dev-only dep için prod severity atama

---

## Handoff

Açılan item'lar normal pipeline'a girer. Critical bug'lar FastTrack edilebilir (analyzer'ı atla, direkt proposer'a):

```bash
stage.sh complete BUG-XXX analyzer --notes "auto-generated by dep-watchdog, details in CVE-XXX"
```
