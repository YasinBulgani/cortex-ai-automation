Veri Yükleme

Veri oluşturma sürecinin ilk adımı, veritabanı şemanızı platforma yüklemektir.

Desteklenen Format Türleri

Tip Açıklama Uzantı

Şema Yükleme Adımları

Adım 1: Dosya Seçin

Yükleme alanına şema dosyanızı sürükleyip bırakın veya dosya seçim düğmesine tıklayın.

Adım 2: Dosya Doğrulaması

Sistem dosyanızı otomatik olarak doğrulayacak ve herhangi bir hata varsa rapor edecektir.

Adım 3: Şema Analizi

Dosya başarıyla yüklendikten sonra, şema analizi sayfasına yönlendirileceksiniz.

Uyarı: Dosya boyutu 100MB'ı aşmayın. Daha büyük dosyalar için yöneticiye başvurun.
İpucu: JSON Schema formatı en ayrıntılı seçenektir ve tüm veri türlerini tamamen destekler.

Şema Analizi

Şemanız yüklendikten sonra, sistem otomatik olarak analiz eder ve sütunları tanır.

Şema Analiz Süreci

  • Sütun Tanıma: Sistem sütun adlarından veri türünü anlamaya çalışır
  • Tür Tespiti: Veri türleri otomatik olarak tespit edilir (tamsayı, metin, tarih vb.)
  • İstatistik Hesaplama: Min/max değerler, null oranları ve daha fazlası hesaplanır
  • İlişki Analizi: Tablolar arasındaki ilişkiler tanınır

Sütun Özellikleri

Özellik Açıklama
Veri Tipi Sütunun veri türü (metin, sayı, tarih vb.)
Tekil Tüm değerlerin benzersiz olup olmadığı
Boş Olabilir Null değerlere izin verilip verilmediği
Varsayılan Değer Sütun için varsayılan değer
Kısıtlamalar Min/max uzunluk, format desenleri vb.
Not: Otomatik algılanan veri türlerini ve ayarlarını ihtiyaca göre manuel olarak düzenleyebilirsiniz.

Kural Yapılandırması

Kurallar, oluşturulacak verilerin nasıl görüneceğini kontrol etmek için güçlü bir yoldur.

Kural Türleri

1. Veri Türü Kuralları

Sütunlar için veri türü kısıtlamaları tanımlar.

{ "type": "string", "minLength": 5, "maxLength": 50, "pattern": "^[A-Z][a-z]+$" }

2. İlişki Kuralları

Tablolar arasında referansiyel bütünlüğü sağlar.

{ "type": "foreignKey", "references": "customers.id", "onDelete": "CASCADE" }

3. Doğrulama Kuralları

Oluşturulan verilerin doğrulama kriterlerini belirtir.

{ "type": "validation", "rules": [ {"field": "age", "min": 18, "max": 120}, {"field": "email", "format": "email"} ] }

4. İş Mantığı Kuralları

Bankacılık alanına özgü kurallar uygular.

{ "type": "business", "rules": [ "account_balance >= 0", "transaction_amount <= account_balance", "interest_rate >= 0 AND interest_rate <= 50" ] }

Kural Ekleme

  1. Sütunu seçin
  2. "Kural Ekle" düğmesine tıklayın
  3. Kural türünü seçin
  4. Kural parametrelerini yapılandırın
  5. Kaydedin
Uyarı: Karmaşık kurallar veri oluşturma süresini uzatabilir. Başlayın basit kurallarla.

Veri Oluşturma

Şemanız ve kurallarınız hazır olduğunda, sentetik veri oluşturmaya başlayabilirsiniz.

Oluşturma Parametreleri

Parametre Açıklama Varsayılan
Satır Sayısı Oluşturulacak satır sayısı 1000
Seed Değeri Aynı verileri yeniden oluşturmak için Rastgele
Batch Boyutu Bellek kullanımını kontrol etmek için 10000
Paralel Çalışanlar Veri oluşturma için kullanılacak iş parçacığı sayısı 4
Locale Yerel veri formatı (TR, EN vb.) TR

Oluşturma Süreci

Adım 1: Parametreleri Ayarlayın

Kaç satır veri oluşturmak istediğinizi ve diğer parametreleri belirtin.

Adım 2: Ön İzleme

Oluşturulacak verilerin bir ön izlemesini görmek için "Ön İzle" düğmesine tıklayın.

Adım 3: Oluşturmayı Başlatın

Tüm ayarlar doğru görünüyorsa "Oluştur" düğmesine tıklayın.

Adım 4: Takip Edin

İlerleme çubuğu oluşturmanın ilerleyişini gösterir. İşlem tamamlanana kadar bekleyin.

İpucu: Seed değerini not etmeyi unutmayın. Daha sonra aynı verileri yeniden oluşturmak istiyorsanız bunu kullanabilirsiniz.

Dışa Aktarma Seçenekleri

Oluşturulan verilerinizi çeşitli formatlar ve hedeflere aktarabilirsiniz.

Desteklenen Formatlar

Format Uzantı Uygunluk Boyut
CSV .csv Tüm uygulamalar Orta
JSON .json Web uygulamaları Büyük
Excel .xlsx Office, analiz Orta
Parquet .parquet Big Data, hız Küçük
SQL INSERT .sql Veritabanları Büyük
XML .xml Enterprise sistemler Çok Büyük

Dışa Aktarma Hedefleri

1. Dosya İndirme

Veriler seçilen formatta bilgisayarınıza indirilir.

Avantajlar: Hızlı, güvenli, internete bağlı değil

2. Veritabanı Aktarımı

Veriler doğrudan bir veritabanına yazılır.

Avantajlar: İndirme yok, doğrudan test yapılabilir

Hedeflenen Veritabanılar: PostgreSQL, MySQL, SQL Server, Oracle, MongoDB

3. Bulut Depolama

Veriler bulut depolamaya yüklenir.

Desteklenen Hizmetler: AWS S3, Azure Blob, Google Cloud Storage

4. API Endpoint

Veriler REST API aracılığıyla erişilebilir hale getirilir.

Avantajlar: Dinamik erişim, sayfalandırma, filtreleme

Dışa Aktarma Yapılandırması

Adım 1: Format Seçin

Istediğiniz çıktı formatını seçin.

Adım 2: Hedef Seçin

Dosya indirme, veritabanı vb. gibi bir hedef seçin.

Adım 3: Seçenekleri Yapılandırın

Sütun seçimi, sıkıştırma, kodlama vb. için seçenekleri yapılandırın.

Adım 4: Dışa Aktar

"Dışa Aktar" düğmesine tıklayın.

Dışa Aktarma Seçenekleri

Seçenek Açıklama
Sütun Seçimi Hangi sütunların dışa aktarılacağını belirleyin
Sıkıştırma Gzip, Bzip2 veya diğer sıkıştırma formatları
Kodlama UTF-8, UTF-16, ISO-8859-1 vb.
Başlık Satırı CSV ve benzer formatlar için sütun başlıklarını dahil edin
Satır Filtreleme Dışa aktarmadan önce satırları filtreleyin
İstatistikler Dışa aktarılan verilere ait özet istatistikleri ekleyin
İpucu: CSV formatı en uyumlu olup, çoğu uygulamada açılabilir. Büyük veri setleri için Parquet formatı önerilir.

Sık Sorulan Sorular

S: Çok fazla satır veri oluşturabilir miyim?

C: Evet! Platform 1.000.000+ satır veri oluşturabiliyor. Bellek ve disk alanınızda bağlı olarak daha fazla olabilir.

S: Veritabanım için oluşturulan veriler gerçekçi midir?

C: Evet, platform gerçekçi veri dağılımları ve bankacılık kurallarını kullanarak gerçekçi veriler üretir.

S: Oluşturduğum verileri silebilir miyim?

C: Evet, Veri Setleri bölümünden istediğiniz zaman verileri silebilirsiniz.

S: Başka türlerde veriler oluşturabilir miyim?

C: Platform özellikle banka verileri için tasarlanmıştır, ancak özel kurallar aracılığıyla diğer veri türlerini de destekleyebilir.

S: Veri oluşturma işlemi ne kadar hızlı?

C: Bu, veri setinin boyutuna, kuralların karmaşıklığına ve sunucu kaynaklarına bağlıdır. Tipik olarak 1000 satır <1 saniyede oluşturulur.
